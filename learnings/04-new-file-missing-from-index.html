В чому була проблема
  • ✕ index.html не прочитаний перед створенням файлу. Claude прочитав попередню ретроспективу щоб зрозуміти формат — але не прочитав index.html. Тому не побачив що там є список який теж потребує оновлення.
  • ✕ Задача "додати ретроспективу" інтерпретована вузько. Створити файл — так. Але "додати" означає щоб він був доступний користувачу, а це вимагає і оновлення навігації.
Як вирішили системно
  • Читати index.html перед створенням будь-якого нового файлу index.html — єдина точка навігації. Оновлення index є частиною тієї ж задачі що й створення файлу, не окремим кроком. CLAUDE.md → Правила роботи з файлами → index.html
Користувач не повинен перевіряти чи з'явився файл у навігації — це частина роботи Claude. "Додати файл" = створити + зробити доступним.
